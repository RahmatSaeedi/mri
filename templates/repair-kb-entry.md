# مدخل پایگاه دانش تعمیرات — «علامت کوتاه»

> ساختارِ علامت ← علت ← درمان. برای جلد ۱۰ (سرویس، نگه‌داری و تعمیرات).

| میدان | مقدار |
|------|------|
| شناسه | KB-«شماره» |
| زیرسامانه | «آهنربا / برودتی / گرادیان / RF / کنسول / بازسازی / خنک‌کاری» |
| شدت | «بحرانی / مهم / جزئی» |
| نسخه / تاریخ | «—» |

## علامت‌ها
«آنچه کاربر/مهندس مشاهده می‌کند (پیام خطا، آرتیفکتِ تصویر، رفتارِ غیرعادی).»

## علت‌های محتمل (از محتمل‌ترین)
1. «علت ۱»
2. «علت ۲»

## گام‌های عیب‌یابی
1. «بررسی/اندازه‌گیری ۱ → نتیجهٔ موردانتظار»
2. «بررسی/اندازه‌گیری ۲»

## درمان / اقدامِ اصلاحی
«مراحلِ تعمیر یا تعویضِ قطعه. قطعاتِ یدکیِ موردنیاز و احتیاطاتِ ایمنی
(ارجاع به تحلیلِ ریسک).»

## راستی‌آزماییِ پس از تعمیر
«آزمونی که سلامتِ سامانه را پس از تعمیر تأیید می‌کند (ارجاع به `test-procedure.md`).»

## یادداشت‌ها / علتِ ریشه‌ای
«تحلیلِ ریشه‌ای و اقدامِ پیشگیرانه برای جلوگیری از تکرار.»
